# Implementation Plan: EduSpark Dark Design System

**Branch**: `002-eduspark-dark-theme` | **Date**: 2026-05-16 | **Spec**: [spec.md](spec.md)  
**Input**: Feature specification from `specs/002-eduspark-dark-theme/spec.md`

---

## Summary

Migrate the Math Buddy frontend from the existing Dandy's World CSS variable token set to the EduSpark dark design system prototyped in `darktheme/eduspark-design/`. The migration covers: CSS custom properties (oklch palette), typography (Space Grotesk + JetBrains Mono), custom utility classes, and a full visual redesign of all five existing screens plus two new pages (Leaderboard, About). The backend, API contracts, and SessionContext business logic are unchanged. Tailwind CSS v3 is retained; no package upgrades are required.

### Critical implementation notes

**`index.css` structure (Tailwind v3 compatible)**:
```css
@tailwind base;
@tailwind components;
@tailwind utilities;

:root {
  --radius: 0.875rem;
  --background: oklch(0.13 0.02 270);
  --foreground: oklch(0.97 0.01 90);
  /* ... full token set from data-model.md ... */
  --gradient-hero: radial-gradient(...);
  --gradient-primary: linear-gradient(135deg, oklch(0.7 0.24 295), oklch(0.7 0.22 330));
  --glow-primary: 0 0 0 1px oklch(0.7 0.24 295 / 0.4), 0 8px 32px -8px oklch(0.7 0.24 295 / 0.55);
}

@layer base {
  * { border-color: var(--border); }
  html { font-family: var(--font-sans); background-color: var(--background); color-scheme: dark; }
  h1, h2, h3, h4 { font-family: var(--font-display); letter-spacing: -0.03em; }
}

@layer utilities {
  .bg-hero-glow { background-image: var(--gradient-hero); }
  .grid-paper { background-image: linear-gradient(...); background-size: 32px 32px; }
  .text-gradient-primary { background: ...; -webkit-background-clip: text; color: transparent; }
  .ring-soft { box-shadow: 0 1px 2px rgb(0 0 0 / 0.4), 0 0 0 1px color-mix(in oklab, var(--ink) 10%, transparent); }
  .ring-soft-lg { box-shadow: 0 2px 4px rgb(0 0 0 / 0.4), 0 16px 48px -12px oklch(0.7 0.24 295 / 0.35), 0 0 0 1px color-mix(in oklab, var(--ink) 10%, transparent); }
  .glow-primary { box-shadow: var(--glow-primary); }
  .glow-accent { box-shadow: var(--glow-accent); }
  .noise-overlay::before { content: ""; position: absolute; inset: 0; background-image: url("data:image/svg+xml;..."); opacity: 0.04; pointer-events: none; }
  .animate-float-slow { animation: float-slow 6s ease-in-out infinite; }
  .animate-shimmer { background: linear-gradient(90deg, ...); background-size: 200% 100%; animation: shimmer 2.5s linear infinite; }
  .animate-pulse-glow { animation: pulse-glow 2.4s ease-out infinite; }
}

@keyframes float-slow { 0%, 100% { transform: translateY(0); } 50% { transform: translateY(-12px); } }
@keyframes shimmer { 0% { background-position: -200% 0; } 100% { background-position: 200% 0; } }
@keyframes pulse-glow { 0%, 100% { box-shadow: 0 0 0 0 oklch(0.7 0.24 295 / 0.5); } 50% { box-shadow: 0 0 0 14px oklch(0.7 0.24 295 / 0); } }
```

**`dandysWorld.js` migration**: Rewrite `cssVars` keys from `--color-primary` style to `--primary` EduSpark style. Remove `characterAsset` and `bgTileAsset` (no longer used). The `setTheme()` function signature is unchanged.

**SessionContext `navigateTo` action**: Check `SessionContext.jsx` — if no action exists to change `screen` without resetting session, add `navigateTo(screen)` that dispatches `{ type: 'NAVIGATE', screen }` and only updates `state.screen`. This is needed so clicking nav links doesn't wipe question/answer state.

**`AppHeader.jsx` hamburger pattern**:
```jsx
const [menuOpen, setMenuOpen] = useState(false)
const handleNav = (screen) => { navigateTo(screen); setMenuOpen(false) }
// Resize listener closes menu at desktop breakpoint
useEffect(() => {
  const close = () => { if (window.innerWidth >= 768) setMenuOpen(false) }
  window.addEventListener('resize', close)
  return () => window.removeEventListener('resize', close)
}, [])
```
